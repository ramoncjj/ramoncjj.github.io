本地 hugo Link to heading

  • 下载

https://github.com/gohugoio/hugo/releases

  • 配置

解压之后能看到hugo.exe文件，将其的绝对路径添加到path后，我们便可全局使用hugo命令

  • 使用
@ECHO OFF
hugo new site demo #创建博客项目demo
cd demo
cd themes #进入themes文件夹
# 下载主题，这里选择hugo-coder，主题下载到themes文件夹里
git clone https://github.com/luizdepra/hugo-coder.git 

#将hugo-coder/exampleSite下的config.toml文件复制并且替换democonfog.toml
cd hugo-coder
cd exampleSite
move config.toml ../
cd ..
move config.toml ../
cd ..
move config.toml ../
cd ..

# 回到demo路径下，开启本地预览
hugo server

PAUSE
# 复制代码保存为bat文件，双击执行

http://localhost:1313/

部署 github Link to heading

  • 将demo 转化为git仓库
cd demo
git init
  • 在demo目录下创建

.github/workflows/main.yml

这是github工作流文件，直白的说有了该文件github会自动给你做一些事情，

下面是文件内容

name: GitHub Pages

on:
  push:
    branches:
      - main  # 触发时机，当你push到main时触发该工作流
  pull_request:

jobs:
  deploy:
    runs-on: ubuntu-20.04
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true  # Fetch Hugo themes (true OR recursive)
          fetch-depth: 0    # Fetch all history for .GitInfo and .Lastmod
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: '0.85.0'

      - name: Build #编译
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3 
        with:
          #你的个人token，没有该token将无法公开到别的仓库，获取方式稍后解释
          PERSONAL_TOKEN : ${{ secrets.blogtoken }} 
          publish_dir: ./public #公开的文件夹
          #公开到哪个仓库，稍后需建立该仓库
          external_repository: ramoncjj/ramoncjj.github.io 
          publish_branch: main  #要公开的branch
          commit_message: ${{ github.event.head_commit.message }}
  • 推送这个仓库到github上，设为私有，接下来为这个仓库设置 Actions secrets

  • Profile Photo -> Settings -> Developer settings -> Personal access tokens -> Generate new token

  • 记得要勾选下面两个scope:
    workflow and write:packages.

  • 创建了token之后复制存好（明文只显示一遍），这个token是账号级别的，接下来把它设置到这个仓库的的Actions secrets里

  • 仓库 -> Settings -> Secrets -> Actions -> New repository secret

    粘贴刚才生成并且保存的token明文 到value框中，并命名，记下这个名字。
    再将这个名字写在main.yml文件中
    PERSONAL_TOKEN : ${{ secrets.<名字>}}

  • 再将main.yml的改动同步到github，设置Action secrets 完毕

  • 创建新仓库

  • [你的 github username].github.io（必须是这个仓库名字）

  • 将这个仓库名填写到上面的main.yml 中

external_repository: ramoncjj/ramoncjj.github.io

  • setting -> pages
    source: Deploy from a branch
    Branch: main -> save

  • 再将main.yml的改动同步到github

  • 部署完毕，访问 https://[你的 github username].github.io/
